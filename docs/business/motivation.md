# Motivation and mission

## Short motivation

PNGtoHTMLapp exists because recreating screenshots, documents and UI references by hand is slow, repetitive and frustrating. The goal is to turn visual input into usable HTML/CSS faster, while keeping the output editable, inspectable and honest about uncertainty.

## Founder motivation

The motivation is to build an AI tool that actually does the boring work developers and builders face every day: taking something visual and converting it into structured web code.

Many AI demos look impressive, but the output is often difficult to use. Visual HTML should be different. It should generate something that can be opened, checked, refined, downloaded and improved. The product should help a real person move faster, not just produce a flashy demo.

## Slovak founder story

Chcem vytvorit nastroj, ktory realne setri cas pri prerabani screenshotov, dokumentov a UI obrazoviek do HTML. Nie iba dalsi AI chat, ale prakticky workflow: nahram obrazok, dostanem preview, viem ho upravit, stiahnut a pouzit.

Najvacsia motivacia je odstranit nudnu manualnu pracu: kopirovanie layoutu, tabuliek, spacingu, textov, formularov a dokumentov. AI ma pomoct cloveku rychlejsie sa dostat k pouzitelnemu vysledku.

Produkt ma byt poctivy. Ak OCR nevie precitat hodnotu, nema si ju vymyslat. Ak je vystup neisty, ma to povedat. Ak ide o dokument alebo vypis, ma respektovat A4, tabulky, print styl a realnu strukturu.

## Mission

Make visual-to-HTML reconstruction fast, practical and trustworthy.

## Product principles

### 1. Faithful reconstruction

Output should preserve visual structure, spacing, hierarchy, tables, typography and document layout as closely as practical.

### 2. Editable standards

The result should be normal HTML/CSS whenever possible. Users should not be locked into a proprietary editor.

### 3. Transparent uncertainty

When OCR or layout interpretation is uncertain, the product should mark it clearly instead of inventing data.

### 4. Fast iteration

Users should be able to refine the output with natural language instructions and immediately preview the result.

### 5. Privacy awareness

Screenshots and documents can contain sensitive information. The product should support safe handling, clear retention policy, deletion after processing and BYOK/private options.

### 6. Practicality over spectacle

The goal is not to impress once. The goal is to produce code people can use repeatedly.

## Why now

This product is possible now because:

- vision and OCR models are strong enough to understand screenshots,
- inference cost is becoming manageable,
- developers already use AI assistants daily,
- teams want faster prototyping and rebuilding workflows,
- many businesses still have visual-only references, screenshots, PDFs and legacy layouts,
- standalone HTML remains universal and easy to inspect.

## Emotional motivation

The best version of the product gives the user relief: they upload a messy screenshot and get a structured starting point instead of staring at a blank file. That feeling matters. The tool should make the user feel faster, more capable and less stuck.

## Long-term vision

Visual HTML can become a visual reconstruction layer for the web:

- screenshot to HTML,
- document image to printable HTML,
- UI reference to component draft,
- legacy page screenshot to modern layout,
- batch conversion and QA comparison,
- API for teams that need this workflow at scale.
